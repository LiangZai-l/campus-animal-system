# 学习路线指南 — 校园流浪动物图鉴系统

> 从环境搭建到完整理解，七步走通一个 Spring Boot + Vue 3 全栈项目。

---

## 学习地图

```
第 1 步    第 2 步        第 3 步        第 4 步        第 5 步        第 6 步      第 7 步
环境搭建 → 项目骨架 →    分层与数据 →   核心业务 →   前后端对接 →   前端组件 →   复盘巩固
          (common       (entity/mapper  (service      (controller    (Vue组件/      (看文档
           security      dto/vo)        实现)          + Axios       路由/状态)     查漏补缺)
           config)                                    + 页面)
```

## 第 1 步：环境搭建（先跑起来）

**目标：** 前后端都能正常启动，浏览器能看到页面。

**阅读：** `环境搭建指南.md`

**动手：**
1. 确认数据库 `campus_animal` 已创建，三张表已建好
2. 启动后端：`cd campus-animal-server && mvn spring-boot:run`
3. 启动前端：`cd campus-animal-web && npm install && npm run dev`
4. 浏览器访问 `http://localhost:5173`，看到登录页 → 第一步完成

**遇到问题：** 翻 `环境搭建指南.md` 第十章"常见问题排查"。

## 第 2 步：项目骨架（基础设施层）

**目标：** 理解系统的"地基"— 统一响应、异常处理、认证授权、系统配置。

**阅读：** `代码学习指南.md` 第二、三、四章

**源码阅读顺序（按依赖关系）：**

```

  Result.java               ← 统一响应体泛型封装（10 分钟）
  BusinessException.java    ← 自定义业务异常（5 分钟）
  GlobalExceptionHandler.java ← 全局异常拦截（15 分钟）

第 2 组：认证授权体系
  TokenUser.java            ← JWT 用户主体（5 分钟）
  JwtTokenProvider.java     ← 令牌生成与解析（20 分钟，重点）
  JwtAuthenticationFilter.java ← 请求拦截与认证（20 分钟，重点）
  SecurityUtils.java        ← 获取当前用户工具（5 分钟）

第 3 组：系统配置
  SecurityConfig.java       ← 安全策略总控（20 分钟，重点）
  MybatisPlusConfig.java    ← 分页插件注册（5 分钟）
  WebMvcConfig.java         ← 静态资源映射（5 分钟）

```

**验证理解：** 能用自己的话解释"一个请求从 Tomcat 到 Controller 经过了哪些过滤器？JWT 是如何被解析和验证的？"

## 第 3 步：分层架构 + 数据层

**目标：** 理解 Entity → Mapper → DTO → VO 的区别，看懂数据如何流转。

**阅读：** `代码学习指南.md` 第五、六章 + `框架理解指南.md` 第五章

**源码阅读顺序：**

```
Entity（数据实体 — 映射数据库表）
  User.java     ← 5 分钟
  Animal.java   ← 5 分钟
  CheckIn.java  ← 5 分钟

Mapper（数据访问 — BaseMapper 内置 CRUD）
  UserMapper.java / AnimalMapper.java / CheckInMapper.java  ← 10 分钟

DTO（入参 — 前端传过来的数据）
  LoginDTO.java → RegisterDTO.java → AnimalSaveDTO.java
  → AnimalQueryDTO.java → CheckInSaveDTO.java  ← 15 分钟

VO（出参 — 返回给前端的数据）
  LoginVO.java → UserVO.java → AnimalVO.java
  → AnimalDetailVO.java → CheckInVO.java  ← 15 分钟
```

**核心理解：**
- **为什么不是直接用 Entity 接收请求和返回响应？** 因为 DTO 带校验注解、VO 不含密码、VO 可跨表聚合。
- **AnimalDetailVO 为什么继承 AnimalVO？** 列表不加载打卡数据（省流量），详情才加 timeline。

**验证理解：** 能画出 `请求数据(DTO) → Entity → 数据库 → Entity → 响应数据(VO)` 的转换流程。

## 第 4 步：核心业务（Service 层）

**目标：** 理解所有业务逻辑的写法。

**阅读：** `代码学习指南.md` 第七章

**重点文件（按重要性排列）：**

```
1. AuthServiceImpl.java     ← 注册 + 登录（30 分钟，理解 BCrypt + JWT 生成）
2. AnimalServiceImpl.java   ← CRUD + 分页 + N+1 优化（40 分钟，重点）
3. CheckInServiceImpl.java  ← 打卡 + 批量查询（20 分钟）
4. FileServiceImpl.java     ← 文件上传安全设计（15 分钟）
```

**AuthServiceImpl 重点关注：**
- 注册时为什么角色固定为 USER？（防越权注册管理员）
- 登录失败为什么统一提示"用户名或密码错误"？（防用户名枚举）
- BCrypt 的 `encode()` 和 `matches()` 分别做了什么？

**AnimalServiceImpl 重点关注：**
- `page()` 方法的 LambdaQueryWrapper 动态查询条件构建
- `getDetail()` 方法的批量查询避免 N+1（这是面试高频考点）

**CheckInServiceImpl 重点关注：**
- `toVOList()` 中的批量查询模式（与 AnimalServiceImpl 一致，看懂了就是掌握了）

**验证理解：** 能在 Knife4j 文档中调试接口，然后对照源码理解每一步。

## 第 5 步：前后端对接（Controller + Axios + 页面）

**目标：** 理解前端如何调用后端接口，数据如何展示。

**阅读：** `代码学习指南.md` 第八、九章

**追踪一条完整的请求链路：**

```
前端操作：用户在首页搜索"小花"，点击搜索按钮

1. HomeView.vue
   fetchList() → api.get('/animals', { params: { name: '小花', page: 1, size: 12 } })

2. api/index.js（Axios 拦截器）
   请求拦截器 → 从 localStorage 取 token → 加到 Authorization 头
   GET /api/animals?name=小花&page=1&size=12
   Authorization: Bearer eyJhbGci...

3. vite.config.js（Vite 代理）
   /api 前缀 → 转发到 localhost:8080

4. SecurityConfig → JwtAuthenticationFilter
   提取 token → 解析 JWT → 存入 SecurityContextHolder

5. AnimalController.page(query)
   @GetMapping → 参数自动绑定 AnimalQueryDTO → 调用 animalService.page()

6. AnimalServiceImpl.page()
   LambdaQueryWrapper → like(Animal::getName, "小花") → selectPage()

7. MyBatis-Plus 分页插件
   拦截 SQL → 改写为 SELECT COUNT(*) + SELECT ... LIMIT 0,12

8. 数据库查询 → 返回数据

9. 原路返回
   Entity → VO → Result.success() → Jackson JSON 序列化 → HTTP 200

10. api/index.js（响应拦截器）
    code === 200 → 返回 data 字段给组件

11. HomeView.vue
    animals.value = res.data.records → 模板重新渲染 → 用户看到卡片列表
```

**对照源码走一遍：** 打开 Knife4j → 用 `/api/animals` 接口 → 打开 `AnimalController.java` → 追进去看 `AnimalServiceImpl.page()` → 理解 MyBatis-Plus 分页 → 回头看前端如何消费数据。

**验证理解：** 能从头到尾画出这条链路。

## 第 6 步：前端组件（Vue 3 生态）

**目标：** 理解 Vue 3 Composition API + 组件化 + 路由 + 状态管理。

**阅读：** `代码学习指南.md` 第十章 + `框架理解指南.md` 第四章

**阅读顺序（由底层到上层）：**

```
第 1 组：基础设施
  main.js          ← 应用初始化 + 插件注册（5 分钟）
  style.css        ← CSS 变量主题定制（10 分钟）
  vite.config.js   ← 代理配置原理（5 分钟）

第 2 组：数据与路由
  api/index.js     ← Axios 封装 + 拦截器（15 分钟，重点）
  stores/auth.js   ← Pinia 状态管理 + 持久化（15 分钟，重点）
  router/index.js  ← 路由配置 + 导航守卫（15 分钟，重点）

第 3 组：布局组件
  App.vue          ← 根组件骨架（5 分钟）
  AppHeader.vue    ← 导航栏（10 分钟）

第 4 组：业务组件
  AnimalCard.vue       ← 动物卡片（10 分钟）
  CheckInTimeline.vue  ← 打卡时间轴（10 分钟）

第 5 组：页面（按复杂度）
  LoginView.vue      ← 登录（10 分钟）
  RegisterView.vue   ← 注册（5 分钟）
  HomeView.vue       ← 首页（15 分钟，重点）
  AnimalDetailView.vue ← 详情（10 分钟）
  CheckInCreateView.vue ← 发布打卡（10 分钟）
  AnimalFormView.vue  ← 新增/编辑表单（20 分钟，重点—复用模式）
  AdminDashboard.vue  ← 管理后台（10 分钟）
```

**核心理解：**
- `ref` vs `reactive`：基本类型用 ref，对象用 reactive
- `computed`：计算属性，依赖变化自动重新计算
- Props 单向数据流：父组件传数据给子组件，子组件不能修改 props
- Pinia vs localStorage：Pinia 是运行时状态，localStorage 是持久化存储
- 导航守卫的 meta 设计：`meta.auth` / `meta.admin` / `meta.guest`

**验证理解：** 能解释"用户登录后刷新页面，为什么还是登录状态？"（Pinia 初始化时从 localStorage 恢复 token → Axios 拦截器自动带 token → 路由守卫检查 token 存在 → 放行）。
