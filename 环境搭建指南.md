## 二、JDK 21 安装与配置

### 2.1 下载 JDK 21

访问 [Oracle JDK 21](https://www.oracle.com/java/technologies/downloads/#java21) 或 [OpenJDK 21](https://adoptium.net/) 下载对应系统的安装包。

Windows 用户推荐下载 `.msi` 或 `.zip` 版本，直接安装到 `D:\JAVA\Develop\jdk` 目录。

### 2.2 配置环境变量

**Windows：**

1. 打开"系统属性" → "高级" → "环境变量"
2. 新建系统变量 `JAVA_HOME`，值为 `D:\JAVA\Develop\jdk`
3. 在 `Path` 变量中添加 `%JAVA_HOME%\bin`
4. 新建系统变量 `CLASSPATH`，值为 `.;%JAVA_HOME%\lib;%JAVA_HOME%\lib\tools.jar`

**macOS / Linux：**

```bash
# 在 ~/.bashrc 或 ~/.zshrc 中添加
export JAVA_HOME=/path/to/jdk-21
export PATH=$JAVA_HOME/bin:$PATH
```

### 2.3 验证安装

```bash
java -version
# 输出示例：
# openjdk version "21.0.2" 2024-01-16 LTS
# OpenJDK Runtime Environment (build 21.0.2+13-LTS)
# OpenJDK 64-Bit Server VM (build 21.0.2+13-LTS, mixed mode, sharing)

javac -version
# javac 21.0.2
```

## 三、Maven 安装与配置

### 3.1 下载 Maven

从 [Maven 官网](https://maven.apache.org/download.cgi) 下载 `apache-maven-3.9.x-bin.zip`。

解压到 `D:\JAVA\apache-maven-3.9.14`。

### 3.2 配置环境变量

1. 新建 `MAVEN_HOME` 系统变量，值为 `D:\JAVA\apache-maven-3.9.14`
2. 在 `Path` 中添加 `%MAVEN_HOME%\bin`

### 3.3 配置 Maven 镜像（加速依赖下载）

编辑 `{MAVEN_HOME}/conf/settings.xml`，在 `<mirrors>` 中添加：

```xml
<mirror>
  <id>aliyun</id>
  <mirrorOf>central</mirrorOf>
  <name>阿里云公共仓库</name>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```

### 3.4 验证安装

```bash
mvn -version
# Apache Maven 3.9.14
# Java version: 21.0.2
```

## 四、MySQL 8.0 安装与配置

### 4.1 安装 MySQL

Windows 用户推荐使用 MySQL Installer：
1. 从 [MySQL 官网](https://dev.mysql.com/downloads/installer/) 下载
2. 选择 "Server only" 或 "Developer Default"
3. 安装过程中设置 root 密码（本项目配置中使用 `meiyoumima`，请根据实际情况修改）
4. 确保 MySQL 服务已启动（服务名：MySQL80）

### 4.2 连接 MySQL

**命令行：**

```bash
mysql -u root -p
# 输入安装时设置的密码
```

**MySQL Workbench：**

1. 打开 MySQL Workbench
2. 点击 "Local instance MySQL80" 连接
3. 输入 root 密码

### 4.3 创建数据库

在 MySQL Workbench 中新建查询，执行：

```sql
CREATE DATABASE campus_animal
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;
```

或者在命令行中：

```bash
mysql -u root -p -e "CREATE DATABASE campus_animal DEFAULT CHARACTER SET utf8mb4 DEFAULT COLLATE utf8mb4_unicode_ci;"
```

## 五、数据库初始化

数据库的建表语句位于 `项目开发方案.md` 的第四章。完整的三张表结构：

### sys_user（用户表）

```sql
CREATE TABLE sys_user (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
    password    VARCHAR(255) NOT NULL COMMENT '密码（BCrypt加密）',
    real_name   VARCHAR(50)  COMMENT '真实姓名',
    role        VARCHAR(20)  NOT NULL DEFAULT 'USER' COMMENT '角色：ADMIN/USER',
    phone       VARCHAR(20)  COMMENT '手机号',
    avatar_url  VARCHAR(255) COMMENT '头像URL',
    status      TINYINT      DEFAULT 1 COMMENT '1-正常 0-禁用',
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户表';
```

### animal（动物档案表）

```sql
CREATE TABLE animal (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    name            VARCHAR(50)  NOT NULL COMMENT '暂定名字',
    type            VARCHAR(30)  NOT NULL COMMENT '动物类型：猫/狗/鸟/其他',
    area            VARCHAR(100) NOT NULL COMMENT '常驻区域',
    description     TEXT         COMMENT '特征描述',
    cover_image     VARCHAR(255) COMMENT '封面照片URL',
    status          TINYINT      DEFAULT 1 COMMENT '1-在校 0-已离校',
    feeder_count    INT          DEFAULT 0 COMMENT '累计投喂次数',
    created_by      BIGINT       COMMENT '创建人ID',
    created_at      DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_name (name),
    INDEX idx_type (type),
    INDEX idx_area (area)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='动物档案表';
```

### check_in（打卡动态表）

```sql
CREATE TABLE check_in (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    animal_id   BIGINT       NOT NULL COMMENT '关联动物ID',
    user_id     BIGINT       NOT NULL COMMENT '打卡人ID',
    content     TEXT         NOT NULL COMMENT '打卡内容',
    images      VARCHAR(1000) COMMENT '打卡图片（多个逗号分隔）',
    mood        VARCHAR(20)  COMMENT '心情：happy/neutral/worried',
    location    VARCHAR(100) COMMENT '打卡地点',
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_animal_id (animal_id),
    INDEX idx_user_id (user_id),
    INDEX idx_created_at (created_at),
    FOREIGN KEY (animal_id) REFERENCES animal(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES sys_user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='打卡动态表';
```

> **提示：** 管理员账号 `admin / admin123` 由后端 `DataInitializer` 在首次启动时自动创建，**不需要手动插入**。

## 六、IDE 配置

### 6.1 IntelliJ IDEA（后端开发）

1. **安装插件：** Lombok（Settings → Plugins → 搜索安装）
2. **导入项目：**
   - File → Open → 选择 `campus-animal-server` 目录
   - IDEA 会自动识别 Maven 项目并下载依赖
3. **配置 JDK：**
   - File → Project Structure → Project → SDK 选择 JDK 21
4. **配置 Maven：**
   - File → Settings → Build, Execution, Deployment → Build Tools → Maven
   - Maven home path 选择 `D:\JAVA\apache-maven-3.9.14`

### 6.2 VS Code / Trae CN（前端开发，可选）

1. 安装 Vue Language Features (Volar) 插件
2. 打开 `campus-animal-web` 目录
3. 终端执行 `npm install`

## 七、后端项目启动

### 7.1 确认配置

打开 `application.yml`，确认以下配置与你本地环境一致：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/campus_animal?...
    username: root
    password: meiyoumima    # ← 改为你的 MySQL root 密码
```

### 7.2 首次启动（自动建管理员）

```bash
cd campus-animal-server
mvn spring-boot:run
```

首次启动时，`DataInitializer` 会检测并自动创建管理员：
```
管理员账号已初始化: admin / admin123
```

### 7.3 验证启动成功

控制台输出 Spring Boot Banner 和日志，无报错：

```
Started CampusAnimalApplication in X.XXX seconds
```

访问 [http://localhost:8080/doc.html](http://localhost:8080/doc.html) 查看 Knife4j API 文档。

## 九、完整流程验证

按以下步骤验证整个系统可正常运行：

1. **启动后端** `mvn spring-boot:run`（端口 8080）
2. **启动前端** `npm run dev`（端口 5173）
3. **注册账号**：访问 `http://localhost:5173/register`，注册一个新用户
4. **登录**：用注册的账号登录，应跳转到首页
5. **管理员登录**：用 `admin / admin123` 登录
6. **新增动物**：导航到"管理" → "新增动物"，填写信息并提交
7. **查看首页**：切换到"首页"，应能看到刚添加的动物卡片
8. **查看详情**：点击动物卡片，进入详情页
9. **发布打卡**：在详情页点击"我来打卡"，填写内容并提交
10. **查看时间轴**：返回详情页，打卡记录应出现在时间轴中

## 十、常见问题排查

### Q1: `mvn` 命令找不到

**原因：** Maven 环境变量未配置。

**解决：** 检查 `MAVEN_HOME` 和 `Path` 变量是否正确设置，重新打开命令行窗口。

### Q2: 项目启动报 `Communications link failure`

**原因：** MySQL 服务未启动。

**解决：**
```bash
# Windows
net start MySQL80

# macOS
brew services start mysql
```

### Q3: 项目启动报 `Unknown database 'campus_animal'`

**原因：** 数据库未创建。

**解决：** 执行第四章的 `CREATE DATABASE` 语句。

### Q4: 项目启动报 `Table 'campus_animal.sys_user' doesn't exist`

**原因：** 表未创建。

**解决：** 执行第五章的建表 SQL。

### Q5: 前端页面空白 / Element Plus 组件无样式

**原因：** `element-plus/dist/index.css` 未导入。

**解决：** 检查 `src/main.js` 中是否有 `import 'element-plus/dist/index.css'`。

### Q6: 登录后刷新页面就退出

**原因：** `localStorage` 中 token 丢失。

**解决：** 检查 Pinia auth store 是否正确持久化了 token（`localStorage.setItem('token', ...)`）。

### Q7: Knife4j 文档页面打不开 (`/doc.html` 404)

**原因：** 可能是 SecurityConfig 中未放行 Knife4j 路径。

**解决：** 确认 `SecurityConfig.java` 中包含：
```java
.requestMatchers("/doc.html", "/swagger-ui/**", "/v3/api-docs/**", "/webjars/**").permitAll()
```

### Q8: 文件上传后访问图片 404

**原因：** 上传目录未正确配置或 `WebMvcConfig` 未正确映射。

**解决：**
1. 确认 `application.yml` 中 `file.upload-dir: ./uploads`
2. 确认项目根目录下存在 `uploads` 目录
3. 确认 `WebMvcConfig` 配置了 `/api/files/**` 的静态资源映射

### Q9: `npm install` 报错

**解决：**
```bash
# 清除缓存后重试
npm cache clean --force
npm install
```

### Q10: 前端 API 请求 404

**原因：** Vite 代理未配置。

**解决：** 确认 `vite.config.js` 中的 proxy 配置正确，且后端在 8080 端口运行。
