## 一、项目概述

**项目名称：** 基于 Spring Boot 3 的校园流浪动物图鉴与动态打卡系统

**学习目标：**
- 深入理解 Spring Boot 3.x 核心框架的设计思想与依赖注入（IoC）机制
- 熟练应用 JDK 21 新特性（Record、Pattern Matching、Virtual Threads 等）

- 掌握 RESTful API 规范的后端接口设计方法
- 掌握文件上传功能实现
- 掌握标准分层架构（Controller → Service → Mapper）
- 培养独立分析业务需求、设计数据库表结构、解决多表关联查询的能力

**用户角色：**
| 角色 | 职责 |
|------|------|
| 学生/教工（USER） | 浏览流浪动物信息，偶遇时发布打卡动态 |
| 系统管理员（ADMIN） | 建立、更新、维护流浪动物电子档案 |

## 四、数据库设计

**数据库名：** `campus_animal`（字符集 utf8mb4）

### 4.1 系统用户表 `sys_user`

```sql
CREATE TABLE sys_user (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50)  NOT NULL UNIQUE COMMENT '用户名',
    password    VARCHAR(255) NOT NULL COMMENT '密码（BCrypt加密）',
    real_name   VARCHAR(50)  COMMENT '真实姓名',
    role        VARCHAR(20)  NOT NULL DEFAULT 'USER' COMMENT '角色：ADMIN/USER',
    phone       VARCHAR(20)  COMMENT '手机号',
    avatar_url  VARCHAR(255) COMMENT '头像URL',
    status      TINYINT      DEFAULT 1 COMMENT '1-正常 0-禁用',
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统用户表';
```

### 4.2 动物档案表 `animal`（核心表）

```sql
CREATE TABLE animal (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    name            VARCHAR(50)  NOT NULL COMMENT '暂定名字',
    type            VARCHAR(30)  NOT NULL COMMENT '动物类型：猫/狗/鸟/其他',
    area            VARCHAR(100) NOT NULL COMMENT '常驻区域',
    description     TEXT         COMMENT '特征描述',
    cover_image     VARCHAR(255) COMMENT '封面照片URL',
    status          TINYINT      DEFAULT 1 COMMENT '1-在校 0-已离校',
    feeder_count    INT          DEFAULT 0 COMMENT '累计投喂次数',
    created_by      BIGINT       COMMENT '创建人ID',
    created_at      DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_name (name),
    INDEX idx_type (type),
    INDEX idx_area (area)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='动物档案表';
```

### 4.3 打卡动态表 `check_in`

```sql
CREATE TABLE check_in (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    animal_id   BIGINT       NOT NULL COMMENT '关联动物ID',
    user_id     BIGINT       NOT NULL COMMENT '打卡人ID',
    content     TEXT         NOT NULL COMMENT '打卡内容',
    images      VARCHAR(1000) COMMENT '打卡图片（多个逗号分隔）',
    mood        VARCHAR(20)  COMMENT '心情：happy/neutral/worried',
    location    VARCHAR(100) COMMENT '打卡地点',
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_animal_id (animal_id),
    INDEX idx_user_id (user_id),
    INDEX idx_created_at (created_at),
    FOREIGN KEY (animal_id) REFERENCES animal(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES sys_user(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='打卡动态表';
```

### 4.4 实体关系图

```
┌──────────┐        ┌──────────────┐        ┌──────────┐
│ sys_user │ 1 ── N │   check_in   │ N ── 1 │  animal  │
│  (用户)   │        │   (打卡动态)  │        │ (动物档案)│
└──────────┘        └──────────────┘        └──────────┘
```

**核心关系：** 一个用户可以发布多条打卡 → 一个动物可以被多次打卡 → 形成"用户 → 打卡 ← 动物"的星型关联。

## 六、核心 API 设计（RESTful）

所有接口统一前缀 `/api`，返回统一格式 `Result<T>`：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

### 6.1 认证模块 `/api/auth`（公开）

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/auth/register` | 用户注册 | 公开 |
| POST | `/api/auth/login` | 用户登录，返回 JWT token | 公开 |

### 6.2 动物档案 `/api/animals`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/animals` | 新增动物（multipart/form-data 含封面图） | ADMIN |
| GET | `/api/animals?name=&type=&page=&size=` | 分页模糊筛选查询 | 登录用户 |
| GET | `/api/animals/{id}` | 查看详情（含打卡时间轴） | 登录用户 |
| PUT | `/api/animals/{id}` | 修改档案（可更换封面） | ADMIN |
| DELETE | `/api/animals/{id}` | 删除档案（级联删除打卡记录） | ADMIN |
| GET | `/api/animals/types` | 获取动物类型枚举列表 | 登录用户 |

### 6.3 打卡动态 `/api/checkins`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/checkins` | 发布打卡（选动物 ID + 填写内容） | 登录用户 |
| GET | `/api/checkins/animal/{animalId}` | 按动物 ID 查询打卡时间轴（时间倒序） | 登录用户 |
| GET | `/api/checkins/my` | 查询当前用户的打卡历史 | 登录用户 |

### 6.4 文件上传 `/api/files`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/files/upload` | 上传图片，返回可访问 URL | 登录用户 |
| GET | `/api/files/{filename}` | 静态访问已上传的图片 | 公开 |

## 十二、开发约定

1. **所有接口返回 `Result<T>` 统一格式**，不直接返回实体
2. **Controller 只做三件事：** 接收参数、调用 Service、返回结果
3. **Service 层放所有业务逻辑**，不直接在 Controller 中写 SQL
4. **数据库操作统一使用 MyBatis-Plus 的 LambdaQueryWrapper**，避免硬编码字段名
5. **敏感信息不返回给前端**（如密码字段加 `@JsonIgnore`）
6. **涉及金钱、安全等关键操作必须有日志记录**
7. **每完成一个阶段，先验证再继续下一阶段**
